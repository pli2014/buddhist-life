## 🎯 文件职责说明

### 根目录文件

#### `index.html`
- **作用**：重定向文件
- **功能**：自动跳转到 `pages/index.html`
- **技术**：使用 meta refresh 实现即时跳转
- **访问**：用户打开网站根目录时首先看到此文件

#### `README.md`
- **作用**：项目主文档
- **内容**：网站介绍、使用说明、技术细节
- **语言**：中文

#### `STRUCTURE.md`
- **作用**：结构说明文档（本文件）
- **内容**：详细的文件组织结构、CSS架构、开发规范

### CSS 文件架构

#### 设计理念
采用 **"全局 + 局部"** 的模块化CSS架构：

1. **全局样式 (style.css)**
   - 提取所有页面共用的样式
   - 定义基础组件和布局
   - 减少代码重复
   - 便于维护和更新

2. **页面样式 ([page].css)**
   - 每个页面独立的配色方案
   - 页面特有的组件样式
   - 覆盖或扩展全局样式
   - 保持页面视觉独特性

#### 加载顺序
```html
<head>
    <!-- 第一步：加载全局样式 -->
    <link rel="stylesheet" href="../css/style.css">
    
    <!-- 第二步：加载页面专用样式 -->
    <link rel="stylesheet" href="../css/[page-name].css">
</head>
```

#### CSS 层叠规则
- 全局样式提供基础样式
- 页面样式可以覆盖全局样式
- 后加载的样式优先级更高
- 使用 specificity 控制精确覆盖

### 页面文件说明

所有页面都遵循统一的HTML结构：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>页面标题</title>
    <link rel="stylesheet" href="../css/style.css">
    <link rel="stylesheet" href="../css/[page].css">
</head>
<body>
    <div class="container">
        <div class="header">...</div>
        <div class="nav-bar">...</div>
        <div class="content">
            <div class="section">...</div>
        </div>
    </div>
</body>
</html>
```

## 🔧 开发规范

### 添加新页面
1. 在 `pages/` 目录创建新的 HTML 文件
2. 在 `css/` 目录创建对应的 CSS 文件
3. HTML 文件中引用两个 CSS 文件
4. 在首页添加导航卡片

### 修改全局样式
1. 编辑 `css/style.css`
2. 所有页面会自动更新
3. 注意测试各页面显示效果

### 修改页面样式
1. 编辑对应的 `css/[page].css`
2. 只影响该页面
3. 不会影响其他页面

### 颜色规范
统一使用金色 `#d4af37` 作为强调色，各页面主题色：
- 首页：深灰蓝 `#2c3e50`
- 健康：棕色 `#8b4513`
- 财富：深蓝 `#1e3a5f`
- 勇气：琥珀 `#92400e`
- 平衡：深紫 `#5b21b6`
- 希望：深红 `#991b1b`
